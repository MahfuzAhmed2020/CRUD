GITHUB -> JENKINS -> DOCKER HUB FLOW
===================================

This project now includes a root Jenkinsfile for CI/CD.

What Jenkins will do on each GitHub push:
1. Checkout the repository from GitHub
2. Run Maven build and tests
3. Build the Docker image from the Dockerfile
4. Tag the image with:
   - latest
   - BUILD_NUMBER-GIT_SHA
5. Log in to Docker Hub using Jenkins credentials
6. Push both tags to Docker Hub

FILES USED
==========
- Jenkinsfile
- Dockerfile
- docker-compose.yml

JENKINS SETUP
=============
1. Install these Jenkins plugins:
   - Pipeline
   - Git
   - GitHub
   - Credentials Binding
2. Make sure Docker is installed on the Jenkins machine or agent
3. Make sure Maven is installed on the Jenkins machine or agent and `mvn` is available on PATH
4. Make sure the Jenkins user can run Docker commands
5. In Jenkins credentials, add:
   - Kind: Username with password
   - ID: dockerhub-credentials
   - Username: your Docker Hub username
   - Password: your Docker Hub password or access token
6. Create a Pipeline job and point it to your GitHub repository
7. Enable GitHub webhook trigger or Poll SCM

GITHUB WEBHOOK
==============
In your GitHub repository:
1. Open Settings -> Webhooks -> Add webhook
2. Payload URL:
   http://YOUR-JENKINS-URL/github-webhook/
3. Content type:
   application/json
4. Choose:
   Just the push event
5. Save the webhook

DOCKER HUB IMAGE NAME
=====================
The Jenkinsfile currently pushes to:
mahfuzdocker20/crud-app

If you want another Docker Hub repo, update this line in Jenkinsfile:
IMAGE_REPO = 'mahfuzdocker20/crud-app'

RUN LOCALLY WITH DOCKER COMPOSE
===============================
From this project folder:

docker-compose up -d
docker ps

To stop:
docker-compose down

PULL A PUBLISHED IMAGE
======================
docker pull mahfuzdocker20/crud-app:latest

or a specific CI build tag:
docker pull mahfuzdocker20/crud-app:<BUILD_NUMBER-GIT_SHA>

TESTER FLOW
===========
Testers only need Docker Desktop.
They do not need Java, Maven, or a local source checkout if they are only pulling the published image.

Example:
docker pull mahfuzdocker20/crud-app:latest

Optional compose override:
set DOCKER_IMAGE=mahfuzdocker20/crud-app:latest
docker-compose up -d
